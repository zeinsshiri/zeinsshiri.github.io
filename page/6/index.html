awtk基础设施搭建

确定PC端哪个系统开发；安装依赖后→下载awtk→运行其下的依赖检测工具→编译（这里是用scons工具）

awtk文件目录结构

应用目录结构

建议使用 AWStudio 中内置的 AWTK Designer 工具新建项目，以上文件以及目录都会 自动生成

源码目录

资源目录

应用运行时将使用 res 目录下的资源，该目录通过打包资源自动生成，如果修改了 design 目录下的资源，则需要重新打包资源让其生效

image目录

AWTK 运行时会优先在 x1、x2、x3 目录中的其中一个查找图片（比如 LCD 的 DPR=1， 则在 x1 目录），如果找不到，则在 xx 目录中查找。对于嵌入式系统，一般只需要 x1 或 xx 的图片。如果开发环境使用高清的 PC 显示器，为了方便 PC 上看效果，建议也准备一套 x2 或 x3 的图片

配置文件

在 AWTK 目录下有个 awtk_config.py 文件（被 SConstruct 引用），在这个文件中可以通 过定义不同含义的宏实现不同的效果；修改 awtk_config.py 文件后，需要重新编译 awtk

渲染模式选择

GL3/ GLES2/ GLES3： 基 于 OpenGL/ GLES 实 现 （nanovg 内 置） ， 由 宏 WITH_NANOVG_GL 决定。适合支持 OpenGL 硬件加速的平台
AGGE：基于 agge 实现。纯软件实现，渲染效果一般，由宏 WITH_NANOVG_AGGE 决定。适合没有 GPU 的嵌入式平台

当 VGCANVAS=’NANOVG’时，对于常用的 AGGE 和 OpenGL 渲染模式又可以细分
1) AGGE-BGR565：表示运行时绘图使用 AGGE 引擎，LCD Frame Buffer 的格式为 BGR565
2) AGGE-BGRA8888：表示运行时绘图使用 AGGE 引擎，帧数据像素格式为 BGRA8888
3) AGGE-MONO：表示运行时绘图使用 AGGE 引擎，LCD Frame Buffer 的格式为 MONO （黑白）

1）OpenGL-GLES2：表示运行时绘图使用 GLES2 引擎，建议在 Windows 平台使用
2）OpenGL-GL3：表示运行时绘图使用 GL3 引擎，建议非在 Windows 平台使用。
