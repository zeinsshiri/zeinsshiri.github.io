前提：图像数字化
图像算法分为三个层次：
1）图像处理：主要是针对图像中的像素级别的数据进行加工。例如：
1.图像增强（增强图像的特征或者提取出某些特定的信息）、
2.图像复原（对由于各种原因导致质量下降的图像进行修复和改善，以尽可能恢复图像的原始质量。这些原因可能包括传感器噪声、压缩失真、光学畸变等）。
2）图像分析：通常是从图像中提取出感兴趣的目标对象，然后对目标进行加工。例如：图像分割、特征提取、特征比对、形状分析、目标跟踪、人脸检测、形态学等，这一类都属于图像分析算法。这一类算法将目标对象提取出来以后，需要有单独的描述方式，不再是底层的仅仅像素表示方式。
3）图像理解：是更高层次的图像的加工算法。例如：压缩感知、3D成像、双目或多目景物重建、场景知识表达等。这一类算法侧重于更高层面的图像的理解和应用。

人的视觉系统

图像处理系统主要由三大部分组成：

  • 图像数字化设备，包括数码相机、数码摄像机或手机等
  • 图像处理设备，包括计算机和存储系统
  • 图像输出设备，包括打印机，也可以输出到Internet上的其他设备

图像数学表示

图像每点都包含了空间，亮度，颜色，时间等信息特征;故每个图像元素都能表示为
$I=f(x,y,z,\lambda,t)；I为光强；xyz为空间坐标；\lambda为波长；t为时间$
图像$I={f(x,y,z,\lambda,t)}$为函数f的集合；
特别地，静止二维单色图像为$I={f(x,y)}$

人眼/传感器输入的图像一般由物体反射光形成，故函数$ I=f(x,y) $可由入射分量和反射分量表示：
$ I=f(x,y)=i(x,y)∗r(x,y);
$$0<i(x,y)<∞ ;$$反射分量 0<r(x,)<1.0 由反射物体的反射率确定$

离散化：

模拟图像：参数是模拟量，典型代表是由光学透镜系统获取的图像，如人物照片、景物照片、图形、X光照片等
离散化实现了计算机对图像的存储/传输/处理
数字图像：模拟图像像素所含信息离散化得到：
1） 采样：图像是二维分布的信息，在 x 轴和 y 轴两个方向上进行n*m次均匀取样（也可根据特定算法进行）；图像分割成了$N\times M$块区域（像素）；像素数量决定了空间分辨率；采样越少，图像清晰度越低，类似马赛克效应

每个像素都有一个二维坐标（整数）（x，y）;而在图像批处理运算时；f(x,y)经常构成一个M×N实数矩阵

2）量化：将连续的波长或光强度信息转换为离散数字表示

  1. 确定量化级别：量化前要确定图像的亮度级别或颜色级别。对于灰度图像，通常使用8位表示，即256个级别；（量化越少，图像的灰度值分割越少，图像对比度越明显，但是会减少图像的细节信息，故对于有大量细节的图像需要较多的灰度级。）
    对于彩色图像的每个通道（如RGB），也使用8位表示。
  2. 线性量化：线性量化是最简单常见的量化方式。它将连续的波长域或光强度范围均匀分割；然后把每个小区间映射到某个整数区间对应量化级别。例如，在灰度图像中，亮度范围从0到255（8位）被分割为256个区间，每个区间对应一个灰度级别。

2.1非统一的图像的采样：
1）在灰度级变化尖锐的区域，用细腻的采样
2）在灰度级比较平滑的区域，用粗糙的采样
3.二进制存储：将映射后的量化级别转换为数字表示，通常使用二进制存储。对于8位表示，使用8个二进制位表示像素的亮度或颜色值

三角函数的正交性：

三角函数系${sin(nx+\frac{k\pi}{2});n,k∈Z}$=${1,\mathrm{cosx,sinx,cos2x,sin2x,\ldots,cosnx,sinnx,~\ldots}}$中任取两元素（任意两个三角函数）:
$f(x)={f(x);x∈[-\pi,\pi]},g(x)={g(x);x∈[-\pi,\pi]}$
两函数内积（即向量各分量的线性组合）$ \lim_{\lambda \to +\infty}\sum_{x=-\pi}^{\pi-\frac{1}{\lambda } } f(x)g(x)\frac{1}{\lambda } $为0;
其所有情况：
$ \begin{aligned}&\int_{-\pi}^{\pi}\sin mx\cdot\cos nxdx:=:0\\&\int_{-\pi}^{\pi}\cos mx\cdot\cos nxdx:=:0\\&\int_{-\pi}^{\pi}\sin mx\cdot\sin nxdx:=:0\end{aligned} $；（n≠m）
if n=m，存在$\int_{-\pi}^{\pi}\cos mx\cdot\cos nxdx:≠:0$⇒故取n≠m情况；
综上：任意两个不同的三角函数在$[-\pi,\pi]$上正交;

周期函数在一个周期上的积分与起点无关：

$要证\int_a^{a+T}f(x)\mathrm{d}x=\int_0^Tf(x)\mathrm{d}x$
⇒$\int_a^{a+T}f(x)\mathrm{~d}x$=$ \int_a^0f(x)\mathrm{d}x+\int_0^Tf(x)\mathrm{d}x+\int_T^{a+T}f(x)\mathrm{d}x
$；要证$\int_a^0f(x)\mathrm{d}+\int_T^{a+T}f(x)\mathrm{d}x=0$
⇒令x=u+T;$\int_a^0f(x)\mathrm{dx}+\int_0^{a}f(u+T)\mathrm{d}u=$$\int_a^0f(x)\mathrm{dx}-\int_a^{0}f(u)\mathrm{d}u=0$

BMP文件格式详解（bmp类）

BMP文件由四个部分构成：

（1）BMP文件头(bmp file header)：固定占用14字节,用于标识文件格式和一些基本信息。

  • 文件类型（2字节）：标识文件为BMP格式，通常为”BM”。
  • 文件大小（4字节）：指定整个BMP文件的大小，以字节为单位。
  • 保留字段（4字节）：保留字段，通常设置为0。
  • 数据偏移量（4字节）：指定图像像素数据相对于文件起始位置的偏移量。

（2）位图信息头(bitmap information)：提供图像数据的尺寸、位平面数、压缩方式、颜色索引等信息

  • 信息头大小（4字节）：指定信息头的大小。
  • 图像宽度（4字节）：图像的宽度，以像素为单位。
  • 图像高度（4字节）：图像的高度，以像素为单位。
  • 颜色平面数（2字节）：总为1。
  • 颜色位深度（2字节）：指定每个像素的位深度，即用于表示颜色的比特数。
  • 压缩类型（4字节）：指定图像数据的压缩类型，常见的有无压缩、RLE压缩等。
  • 图像大小（4字节）：指定图像数据的大小，以字节为单位。若使用无压缩存储，通常为0。
  • 水平分辨率（4字节）：图像的水平分辨率，以像素每米为单位。
  • 垂直分辨率（4字节）：图像的垂直分辨率，以像素每米为单位。
  • 使用的颜色数（4字节）：图像实际使用的调色板中的颜色数目。若为0，则表示使用了所有颜色。
  • 重要颜色数（4字节）：对图像产生重要影响的颜色数目。若为0，则表示所有调色板项都重要。

（3）~~~~调色板~~~~(color palette)：可选，如使用索引来表示图像，调色板就是索引与其对应的颜色的映射表，在使用 256位彩色、16位彩色等情况下用到

（4）位图数据(bitmap data)：存放图像的像素数据；存放顺序从左到右、从下到上的顺序存放。
每一行像素数据需要保证4字节对齐，不对齐的需要补充空字节，这个是BMP文件格式本身要求的。对于RGBA 32位色彩格式来说没有任何问题，但是对于Gray或者RGB格式的数据 ，有时一行像素需要补充一些空字节。

条件概率及其基本推论

.条件概率： 设试验E的样本空间为Ω, A, B是事件,则在B发生条件下A发生的概率, 称条件概率P(A|B)；

如此样本空间从全空间Ω缩小到B；事件A包含的样本点从A缩小到（A∩B）；P(A|B)的概率空间（B,F,P(AB)/P(B)）

A的条件概率与A的全空间概率关系：
设（Ω,F,P）；事件A,B∈事件域F；且P（B）>0;则P（A|B）=P(AB)/P(B)
即AB在B样本中的概率=AB在Ω样本中的概率/B在Ω样本中的概率；

条件概率满足概率公理；足以作为一个论证起点

**⇒乘法定理(交集概率)**：P(AB)=P(A|B)P(B) 前提P(B)>0

⇒全概率公式:由因推果，已知A事件在各原因/前提Ci下发生，由Ci导致A发生的条件概率P(A|Ci)以及因素本身的概率，推断A事件发生的概率（互斥因素Ci的 概率分布和Ci所导致结果A|Ci的条件分布⇒结果A概率)

⇒Bayes公式：由果溯因，已知A事件在各原因/前提Ci下发生，由结果概率P(A)，各因素Bi导致A发生的条件概率P(A|Bi)及Bi本身概率，推断结果A发生是由Bi导致的概率P（Bi|A）（即先验⇒后验；由以往观测结果{互斥因素bi分布P(bi),P(A),条件分布P(A|Bi)}⇒是由各因素导致的概率）
