生成分量无序结构的h元组合(子集/组合问题)

发表于 2024-02-07 更新于 2024-02-08 分类于 数据结构&算法 阅读次数： 本文字数： 3.4k 阅读时长 ≈ 3 分钟

给定分量集：
1）分量集元素可无限次选取2）解空间的解无序结构
求解空间所有子集（也可以说是组合问题）

回溯策略：遍历所有h元组合所有可能
1）求出问题的解空间
2）声明3个容器：
1.路径：问题的解在搜索过程中动态产生
2.分量列表
3.符合约束条件的解
3）遍历分量集选择第一个分量→迭代地遍历分量集选择第二个分量…….直至到达递归终止条件
{每选择一个分量：判定是否满足约束条件
不满足：continue跳至下轮本分量选择的下一轮遍历，相当于放弃了本轮遍历的子树
满足：把路径里的结果push into 结果列表；pop路径；返回上一个分量选择调用
}

分量集无重复元素

因为组合（无序结构）；（5，4）（4，5）是一样的组合，考虑去重
1）对结果列表进行去重。但这个方法效率很低，因为：

  • 当数组元素较多，尤其是当 target 较大时，搜索过程会产生大量的重复子集。
  • 比较子集（数组）的异同非常耗时 ，需要先排序数组，再比较数组中每个元素的异同。

2）优雅的实现方式是生成过程识别到这种情况直接减枝
第一个分量$x_{i}$的子树生成了所有包含$x_{i}$的组合；第一个分量非$x_{i}$的子树遍历{分量集-$x_{i}$}生成的子树可以避免生成包含$x_{i}$的组合

分量集有重复元素

解决思路：规定每个值只能在每个分量位置上出现一次；实现方式比较巧妙：由于数组是已排序的，因此相等元素都是相邻的。这意味着在某轮选择中，若当前元素与其左边元素相等，则说明它已经被选择过，因此直接跳过当前元素。
