暴力搜索/生成h元组合（分量有序结构）

发表于 2024-02-07 更新于 2024-02-08 分类于 数据结构&算法 阅读次数： 本文字数： 3.3k 阅读时长 ≈ 3 分钟

多叉树每层存储的都是分量，枚举h元组合
1）排列问题：枚举h维向量所有可能（分量有序结构，有序对）
2）组合问题：枚举h个分量的无序对所有可能（分量无序结构，无序对）
3）避免无效搜索： 若含约束条件，（1）用约束函数剪去不满足约束的子树；（2）用限界函数剪去得不到最优解的子树。

回溯策略：遍历所有h元组合所有可能
1）求出问题的解空间
2）声明3个容器：
1.路径：问题的解在搜索过程中动态产生
2.分量选择列表 **：当前可选择的分量
3.符合约束条件的解
3）从解空间树根结点出发，以深度优先搜索方式搜索解空间（h轮迭代选取第一个分量，h-1轮迭代选取第二个分量。。。。每轮选取结束后递归开启下轮迭代），直至到达决策树底层，⽆法再做选择的条件 **；
搜索过程中，若判断出某子树不包含问题的解，则跳过对该子树解子空间的生成

排列（分量集含相同元素）

去除重复排列：最直接地，考虑借助一个哈希表，直接对排列结果进行去重（不优雅，要多维护一个哈希表）

生成重复排列的搜索分支是没必要的，应当被提前识别并剪枝，这样可以进一步提升算法效率。

在每轮分量选择中开启一个哈希表 duplicated ，标记该轮中已经尝试过的元素，并将重复元素剪枝，使得每个分量位置上的值只能出现一次

假设元素两两互不相同，则 n 个元素共有 n! 种排列；在记录结果时，需要复制长度为 n 的列表，使用 O(n) 时间。因此时间复杂度为 O(n!n)
