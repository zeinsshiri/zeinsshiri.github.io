    拓扑结构

    多对多的网状结构，由「顶点 vertex」和「边 edge」组成

    序结构

    根据边是否有方向，可分为「无向图 undirected graph」和「有向图 directed graph」ex:follow关系

    根据所有顶点是否连通，可分为「连通图 connected graph」和「非连通图 disconnected graph」

    • 对于连通图，从某个顶点出发，可以到达其余任意顶点。
    • 对于非连通图，从某个顶点出发，至少有一个顶点无法到达。

    可为边添加“权重”变量，从而分为「有权图 weighted graph」ex：亲密度网络

    • 两顶点「邻接 adjacency」：两顶点间存在边相连
    • A 到 B 的「路径 path」： A 到B 经过的边构成的序列
    • 「度 degree」：顶点拥有的边数。对于有向图，「入度 In-Degree」表示有多少条边指向该顶点，「出度 Out-Degree」表示有多少条边从该顶点指出

    数据操作及实现（代数结构）

    基于邻接矩阵adjacency matrix实现

    • 顶点不能与自身相连，因此邻接矩阵主对角线元素没有意义
    • 对于无向图，两个方向的边等价，此时邻接矩阵关于主对角线对称
    • 将邻接矩阵的元素从 1 和 0 替换为权重，则可表示有权图。

    使用邻接矩阵表示图时，我们可以直接访问矩阵元素以获取边，因此增删查操作的效率很高，时间复杂度均为 O(1) 。然而，矩阵的空间复杂度为 O(n2) ，内存占用较多。

    初始化：

    n×n matrix的行（列）坐标代表顶点，矩阵元素代表边， 1 /0 表示两顶点间是否存在边

    1）初始化长度为 n 的顶点列表 vertices存储值O(n)
    2）初始化 n×n 大小的邻接矩阵 ( 二维列表)adjMat 存储边状态（序结构） O(n^2)

    添加顶点O(n)

    1)顶点列表push val
    2)matrix 加多一行0;O(n)
    3）二维列表迭代n行在每个列表后push0;O(n)

    删除顶点：O(n^2)

    在邻接矩阵中删除一行一列。当删除首行首列时达到最差情况，需要将 (n−1)^2 个元素“向左上移动”，从而使用 O(n^2) 时间

    添加或删除边：O(1)

    直接在邻接矩阵中修改指定的边即可，使用 O(1) 时间。而由于是无向图，因此需要同时更新两个方向的边。

    基于邻接表adjacency list实现

    n 个链表表示图。第 i 条链表存储了i顶点的所有邻接顶点（即与该顶点相连的顶点）

    邻接表仅存储实际存在的边，而边的总数通常远小于 n^2 ，因此它更加节省空间。然而，在邻接表中需要通过遍历链表来查找边，因此其时间效率不如邻接矩阵。

    邻接表结构与哈希表中的“链式地址”非常相似，可采用类似方法优化效率。比如当链表较长时，可以将链表转化为 AVL 树或红黑树，从而将时间效率从 O(n) 优化至 O(log⁡n) ；还可以把链表转换为哈希表，从而将时间复杂度降低至 O(1) 。

    设无向图的顶点总数为 n、边总数为 m

    添加顶点： O(1)

    在邻接表中添加一个链表，并将新增顶点作为链表头节点;
    或把顶点*映射到 空 邻节点*列表

    添加边：O(1)

    在顶点对应链表的末尾添加边即可，使用 O(1) 时间。因为是无向图，所以需要同时添加两个方向的边。
    或直接在对应的hashtable[key]添加hash value

    初始化：O(n+m)

    1）定义hash table：key为顶点 ，value为邻节点
    2）遍历边集{（A ，b）}；把连通点的key(A和b) push into hashtable；并指定value为空的Vertex列表
    3）把两个邻节点    *push into corresponding hashtable[key] list

    在邻接表中创建 n个顶点和 2m邻节点，使用 O(n+m) 时间。

    删除边：O(m)

    在顶点对应链表中查找并删除指定边，使用 O(m) 时间。在无向图中，需要同时删除两个方向的边。

    删除顶点：O(n+m) ??????

    1）hash table直接删除这个key的bucket
    2）遍历所有value 列表 in hash bucket，删除包含指定顶点的所有边（即删除list里面的顶点 *），列表实现为O(n+m)

    效率对比

    设图有 n个顶点和 m条边

    1. 邻接表使用 Vertex 节点类表示顶点;使每个顶点都是唯一对象，此时删除顶点时就无须改动其余顶点;而通过顶点值来区分不同顶点，那么值重复的顶点将无法被区分。
    2. 邻接矩阵使用顶点列表索引来区分不同顶点。假设要删除索引为 i的顶点，则需要遍历整个邻接表，将其中 >i的索引全部减 1 ，这样操作效率较低。

    邻接表（哈希表）的时间与空间效率最优。但实际上，在邻接矩阵中操作边的效率更高，只需要一次数组访问或赋值操作即可。综合来看，邻接矩阵体现了“以空间换时间”的原则，而邻接表体现了“以时间换空间”的原则。

    遍历

    常用两种策略：
    1）直接穷举：一笔画问题；关键在于能不能人为找出一种元素间的序结构；然后按层序遍历
    2）分治：遍历起点0的图=访问0+遍历0邻节点（不含0）的图

    广度优先遍历 breadth-first search；O(|V|+|E|)

    依据是数据本身存在层序结构：
    遍历起点到各节点的路径长度（经过边数量），由近到远分层（比如起点的邻节点都是第一层，第一层的领节点路径长度都是2，为第二层，以此类推）；逐层访问，同层的线性访问； 便能一笔画访问

    1）声明存放遍历数据的列表；用于区分层序的队列；记录已访问节点的哈希表 visited
    2）遍历起点startVet入队作为第0层
    3）队首节点出队存至遍历列表和哈希表；线性遍历该节点所有邻节点，依次入队作为第1层节点（起点的邻节点肯定是第1层）；循环这个操作

    时间复杂度： 所有顶点都会入队并出队一次，使用 O(|V|) 时间；在遍历邻接顶点的过程中，由于是无向图，因此所有边都会被访问 2 次，使用 O(2|E|) 时间；总体使用 O(|V|+|E|) 时间。

    空间复杂度： 列表 res ，哈希表 visited ，队列 que 中的顶点数量最多为 |V| ，使用 O(|V|) 空间。

    深度优先遍历depth-first search

    1）声明hash table存储已访问节点，使访问邻节点时可以跳过已访问节点；声明遍历列表存储遍历顶点元素
    2）遍历起点为x的图=访问x+遍历x所有邻节点为起点的图（去除x）
    3）递归实现

    复杂度分析

    时间复杂度： 所有顶点都会被访问 1 次，使用 O(|V|) 时间；遍历时，每有1条边便会访问 2 次邻节点，使用 O(2|E|) 时间；总体使用 O(|V|+|E|) 时间。

    空间复杂度： 列表 res ，哈希表 visited 顶点数量最多为 |V| ，递归深度最大为 |V| ，因此使用 O(|V|) 空间。

    图常见应用

    许多现实系统都可以用图来建模，相应的问题也可以约化为图计算问题
